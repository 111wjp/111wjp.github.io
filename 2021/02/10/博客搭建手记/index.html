博客搭建手记

2021-02-10

前言

这是一篇采用GitHub Pages + Hexo方式搭建个人博客网站的简要手记

前期的准备工作

  • 拥有一个Github 账号
  • 安装node.js
  • 安装git
  • 安装Hexo
    在终端下输入
    npm install -g hexo-cli
    然后检查是否安装成功
    hexo --version

配置本地博客环境

新建一个文件夹用来存放和管理博客文件，我这里就命名为Blog。
在终端下输入以下指令构建基本的本地博客环境


在本目录空白处使用gitbash打开
输入以下指令，来预览本地搭建好的博客网站
hexo s
在浏览器中输入 http://localhost:4000 回车就可以预览效果了。
这是最原始的博客网站，没有自己写的文章，没有个性化的主题，头像，个签等等
所以接下来就配置好想要的东西

安装主题，配置个人信息

在 https://hexo.io/themes 网站上找一个喜欢的主题吧，我用的是[laughing]（https://github.com/BoizZ/hexo-theme-laughing）。
在主题github的README.md文档，作者会告诉你主题的使用方法，以laughing为例，以下是官方配置教程

  1. laughing主题使用pug模板引擎开发，需要安装前置依赖组件。
    npm install hexo-renderer-pug --save
  2. 拷贝主题到themes目录
  3. 修改主程序的_config.yml文件
    theme: hexo-theme-laughing
  4. 在themes\hexo-theme-laughing下修改一些个人信息

    网站图标

SEO

站点标题、副标题、描述沿用站点的配置。

页头

菜单按钮风格目前支持两种：dark，light。

作者信息

作者的名字沿用站点的配置。

内容宽度

可自定义内容的最大显示宽度。

社交信息

主题内置的社交图标目前有：facebook, twitter, weibo, wechat, github, stackoverflow, linkin, email, segmentfault, flickr, zhihu, disqus, douban, bilibili

如需新增请在Issues里面提出

多说

多说是自由开启的，默认开启，如果不需要则在配置文件里面关闭即可。

需要注意的是，多说的url配置默认使用站点的配置。请更改站点配置中url为自己的域名。

部署到GitHub Pages

生成项目

在GitHub右上角点击New repository。Repository name必须是固定格式，为 “你的github用户名.github.io” 。创建完成后你便拥有了一个自己的静态网页

配置 SSH key

用git配置SSH Key，以便连接你的本地Blog环境和GitHub项目

  1. 首先在本地blog文件夹内打开git bash

  2. 执行以下命令全局配置一下本地账户

  3. 然后开始生成密钥 SSH key
    ssh-keygen -t rsa -C '邮箱地址'

  4. 查看密钥
    cat ~/.ssh/id_rsa.pub

  5. 添加主机到本机SSH可信列表
    ssh -T git@github.com
    若返回 Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access. 证明添加成功

  6. 在github上添加密钥
    右上角点击自己的头像，进入setting，在SSH and GPG keys这个选项中New SSH key一下，然后把刚生成的密钥拷贝进去就好了
